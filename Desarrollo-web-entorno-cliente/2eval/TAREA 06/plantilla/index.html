Ejercicios formularios

Para este bloque de actividades vamos a repasar las peticiones HTTP GET a API REST públicas y la implementación de formularios reactivos. Para ello, primeramente deberás crear un componente header y tres vistas: enunciado, ejercicio1 y ejercicio2.

Para cada uno de los dos ejercicios siguientes deberás implementar los servicios, eventos e interfaces que sean necesarios.

Ejercicio 1

Implementa el formulario proporcionado de manera que se permita recoger los datos introducidos por el usuario y, a partir de ellos, realizar una petición HTTP GET a la API REST pública de The Rick & Morty API. De este modo, el usuario podrá eleccionar si quiere realizar una búsqueda por personaje o por capítulo e introducir un nombre. Si el usuario selecciona la búsqueda por personaje, se mostrará el listado de todos los capítulos de la serie en los que aparece el personaje indicado -ejemplo: 'Pilot - S01E01'-. Si la búsqueda se realiza por capítulo, se indicará el nombre del capítulo y se mostrarán todos los personajes que salen en dicho capítulo. Nota: se entenderá que el usuario introduce los nombres correctamente.

Ejercicio 2

Implementa el formulario proporcionado de manera que se permita recoger los datos introducidos por el usuario y, a partir de ellos, realizar una petición HTTP GET a las API REST públicas de API Rest Countries para obtener datos sobre países y API Open Weather Map, para obtener información meteorológica. De este modo, el usuario podrá indicar un país sobre el que obtener información y qué información concreta quiere visualizar. Nota: los nombres de los países se indicarán en inglés y se entenderá que el usuario introduce los nombres correctamente.
